0x01 前言

这次回归到了以往的模式，继续写一些入门向的东西。 这次是如何使用VSCode进行代码托管的介绍。 正好利用一下上次建的Gitea，当然，如果使用Github也是大同小异的。

0x02 环境准备

本文以5.4.33-3-MANJARO,Gitea 1.11.4环境为参考，其他平台请自行修改部分内容。 本文中域名gitea.moe仅在作者的内网使用，实际使用时请自行替换。

  1. VSCode
  2. Git

0x03 仓库准备

获取仓库链接

首先在Gitea上创建一个仓库

然后在上传文件的旁边选择SSH，并复制里面的链接，这就是我们仓库的链接了。

root@localhost:KiraKiseki/Git-Start.git

如果主机名是localhost，就需要在Path/to/gitea/custom/conf/app.ini中修改SSH_DOMAIN和DOMAIN，修改为主机的域名。然后重启Gitea

由于作者的Gitea是以root用户运行的，所以用户名为root。 现在你复制下来的仓库链接应该是这样的。

root@gitea.moe:KiraKiseki/Git-Start.git

本地仓库

现在，在VSCode中打开你准备存放代码的文件夹(本文以GIT为例)。

现在点击左边的“分叉”符号，应该会提示没有git储存库。

!(https://i.loli.net/2020/04/29/RrXFbQfm7Gzv5ON.png)

使用Ctrl+Shift+P 启动命令面板，选择Git:Clone。

然后输入我们上一步获得的仓库链接，回车后选择你刚才打开的工作区。

现在返回目录视图，应该像是这样:

0x04 提交代码

打开一个git bash，输入下面两个命令，配置全局的用户名和邮件。

git config --global user.name "your name"
git config --global user.email "email@example.com"

接下来随便写一个HelloWorld。

这个时候，左边应该显示有更改。

我们再切换到Git视图。

可以看到，这里已经出现了你刚才所做的更改。

在消息一栏中填入提交代码时的附加信息，然后点击总是(个人偏好)。

然后在左下角就会出现这样的图示。

点击循环的图标之后会询问你的用户名和密码，输入之后就会提交代码到Gitea上。

0x05 免密码登录

使用上面的方法每次提交代码都要输入用户名和密码挺烦的，有什么好的方法吗 当然，那就是免密码登录。

生成密钥对

(如果已经生成过，可选择跳过此步)

在终端中运行

ssh-keygen -t rsa -C "email@example.com"

然后一直回车，系统就会为你生成一个密钥对在~/.ssh目录下。

其中id_rsa.pub为公钥，id_rsa为私钥。公钥与私钥有一一对应的关系

免密码连接

将你的公钥中的内容全部复制，并在Gitea的设置中添加你的SSH公钥，like this。

现在来测试ssh连接,将root替换为你运行Gitea的用户名

ssh -T root@gitea.moe

如果返回这样的消息，则代表你成功配置了免密码登录。

现在再试试上文中的提交操作？是不是已经不用输入用户名和密码变得方便了许多？

0x06 后记

这次算是配置好了个存自己写过的屎山的地方吧。以后传代码上去也方便了很多。因为是私有的服务，所以还是挺方便的，也拥有对自己仓库的完全控制权。本人也对Git不甚了解，有些地方难免会出一些纰漏，还望各位能够不吝斧正。 这次写的依旧很仓促，在实际操作中出现了什么问题的话，也可以来联系作者来解决:)

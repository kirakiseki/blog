0x01 前言

窝又来整活了(鞠躬) 这是一篇 真 · 水文 在上一篇文章中我们提到了现在RSS信息源难以寻找和稀少的问题，在这样的背景下，我们本篇文章的主角 RSSHub 便应运而生。

RSSHub 是一个开源、简单易用、易于扩展的 RSS 生成器，可以给任何奇奇怪怪的内容生成 RSS 订阅源。RSSHub 借助于开源社区的力量快速发展中，目前已适配数百家网站的上千项内容 1

通过RSSHub我们就可以订阅各种奇奇怪怪的信息源了。放一些官方文档(需科学上网)的示例吧。

可以看到，这东西是真的恐怖，你想到想不到的都可以在这里找到。就算没有找到你也可以通过他的接口自己编写信息源然后分享出来。

0x02 安装

  • 安装命令部分来自原作者2

首先从Git下载RSSHub的源码

git clone https://github.com/DIYgod/RSSHub.git
cd RSSHub

在下载完成之后安装依赖

npm install

由于在国内访问npm速度很慢，这里我们使用淘宝 NPM 镜像cnpm

cnpm install

在安装了依赖之后，启动服务

npm start

但是这样会在SSH断开后终止进程，因此我们使用pm2工具来启动进程。

pm2 start lib/index.js --name rsshub

看到如图的字样即为启动成功。

将RSSHub加入开机启动

pm2 startup

不要忘记在面板中开放端口。 现在访问https://IP:1200/应该就可以看见默认的页面了。

0x03 反向代理

我们要用端口访问的话，就很不爽，于是我们要想办法去掉这个端口号，改为使用/rsshub的文件夹进行访问。

由于服务器安装的是Nginx，因此我们使用Nginx的反向代理来实现。

由于这里涉及到反向代理的前置技能树，这里就不展开讲述了，这里直接给出验证有效后的配置文件3。 请在站点Nginx配置文件中的server代码块中添加如下配置。

location / {
  proxy_pass http://localhost:1200/;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  proxy_set_header X-Forwarded-Proto $scheme;
  proxy_redirect off;
  proxy_set_header Host $host;
}

在更改了配置文件并重启了Nginx服务之后，我们就可以使用https://RSSHubIP/访问了。

0x04 生成文档

由于docs.rsshub.app被土啬了，所以我们要在本地生成一份文档用作参考。 原作者已经在Github上面提供了一份docs的源文件4，所以我们只要进行生成就好了。 我们可以看见，里面有个.vuepress文件夹。一番搜索之后，发现作者就是使用VuePress5来生成文档的。

由于这里是现有项目，所以我们将VuePress作为本地依赖来安装。

yarn add -D vuepress

在安装好了之后，我们通过yarn来运行命令生成静态HTML文件。

yarn docs:build

静态的HTML文件会生成在.vuepress/dist下面，将文件上传到服务器上就可以看到RSSHub的文档了。

e.g. 2 含参订阅

这里我们以订阅bilibili番剧为例 可以看到，路由中为我们提供了一个必选参数:mediaid，在下文的文档中提到这个id可以从番剧主页获取。

比如我们想订阅番剧某科学的超电磁炮T夹带私货x2 从番剧网页的URL中 https://www.bilibili.com/bangumi/media/md28224095/?from=search&seid=7433280371531657823

很容易得到我们想要的参数mediaid为28224095。

于是我们得到的信息源地址就是 https://RSSHUBIP/bilibili/bangumi/media/28224095 将它作为信息源加入TT-RSS的订阅中。
